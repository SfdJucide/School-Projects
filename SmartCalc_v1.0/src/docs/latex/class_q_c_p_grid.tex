\doxysubsection{Подробное описание}
Responsible for drawing the grid of a \mbox{\hyperlink{class_q_c_p_axis}{QCPAxis}}. 

This class is tightly bound to \mbox{\hyperlink{class_q_c_p_axis}{QCPAxis}}. Every axis owns a grid instance and uses it to draw the grid lines, sub grid lines and zero-\/line. You can interact with the grid of an axis via \mbox{\hyperlink{class_q_c_p_axis_a63f1dd2df663680d2a8d06c19592dd63}{QCPAxis\+::grid}}. Normally, you don\textquotesingle{}t need to create an instance of \mbox{\hyperlink{class_q_c_p_grid}{QCPGrid}} yourself.

The axis and grid drawing was split into two classes to allow them to be placed on different layers (both \mbox{\hyperlink{class_q_c_p_axis}{QCPAxis}} and \mbox{\hyperlink{class_q_c_p_grid}{QCPGrid}} inherit from \mbox{\hyperlink{class_q_c_p_layerable}{QCPLayerable}}). Thus it is possible to have the grid in the background and the axes in the foreground, and any plottables/items in between. This described situation is the default setup, see the \mbox{\hyperlink{class_q_c_p_layer}{QCPLayer}} documentation.

\doxysubsubsection{\texorpdfstring{applyDefaultAntialiasingHint()}{applyDefaultAntialiasingHint()}}
{\footnotesize\ttfamily void QCPGrid\+::apply\+Default\+Antialiasing\+Hint (\begin{DoxyParamCaption}\item[{\mbox{\hyperlink{class_q_c_p_painter}{QCPPainter}} $\ast$}]{painter }\end{DoxyParamCaption}) const\hspace{0.3cm}{\ttfamily [protected]}, {\ttfamily [virtual]}}

This function applies the default antialiasing setting to the specified {\itshape painter}, using the function \mbox{\hyperlink{class_q_c_p_layerable_acb663e375d2d36dc5c55021ee5a2119b}{apply\+Antialiasing\+Hint}}. It is the antialiasing state the painter is put in, when \mbox{\hyperlink{class_q_c_p_grid_aad9a312d998e2d170956334d4cb80be2}{draw}} is called on the layerable. If the layerable has multiple entities whose antialiasing setting may be specified individually, this function should set the antialiasing state of the most prominent entity. In this case however, the \mbox{\hyperlink{class_q_c_p_grid_aad9a312d998e2d170956334d4cb80be2}{draw}} function usually calls the specialized versions of this function before drawing each entity, effectively overriding the setting of the default antialiasing hint.

{\bfseries{First example\+:}} \mbox{\hyperlink{class_q_c_p_graph}{QCPGraph}} has multiple entities that have an antialiasing setting\+: The graph line, fills and scatters. Those can be configured via \mbox{\hyperlink{class_q_c_p_layerable_a4fd43e89be4a553ead41652565ff0581}{QCPGraph\+::set\+Antialiased}}, \mbox{\hyperlink{class_q_c_p_abstract_plottable_a089d6b5577120239b55c39ed27c39536}{QCPGraph\+::set\+Antialiased\+Fill}} and \mbox{\hyperlink{class_q_c_p_abstract_plottable_a2f03f067ede2ed4da6f7d0e4777a3f02}{QCPGraph\+::set\+Antialiased\+Scatters}}. Consequently, there isn\textquotesingle{}t only the \mbox{\hyperlink{class_q_c_p_abstract_plottable_ac032077fb0db93d6faa3273d02363398}{QCPGraph\+::apply\+Default\+Antialiasing\+Hint}} function (which corresponds to the graph line\textquotesingle{}s antialiasing), but specialized ones like \mbox{\hyperlink{class_q_c_p_abstract_plottable_a8d06a59ea23324cce6330ebf2262c0ed}{QCPGraph\+::apply\+Fill\+Antialiasing\+Hint}} and \mbox{\hyperlink{class_q_c_p_abstract_plottable_ac95f26b15a1e5d9c7bd2c0a46d760fc9}{QCPGraph\+::apply\+Scatters\+Antialiasing\+Hint}}. So before drawing one of those entities, \mbox{\hyperlink{class_q_c_p_graph_a2b0849598f06e834b43ce18cd13bcdc3}{QCPGraph\+::draw}} calls the respective specialized apply\+Antialiasing\+Hint function.

{\bfseries{Second example\+:}} \mbox{\hyperlink{class_q_c_p_item_line}{QCPItem\+Line}} consists only of a line so there is only one antialiasing setting which can be controlled with \mbox{\hyperlink{class_q_c_p_layerable_a4fd43e89be4a553ead41652565ff0581}{QCPItem\+Line\+::set\+Antialiased}}. (This function is inherited by all layerables. The specialized functions, as seen on \mbox{\hyperlink{class_q_c_p_graph}{QCPGraph}}, must be added explicitly to the respective layerable subclass.) Consequently it only has the normal \mbox{\hyperlink{class_q_c_p_abstract_item_a5579ce9ec7cad202499886b042448b10}{QCPItem\+Line\+::apply\+Default\+Antialiasing\+Hint}}. The \mbox{\hyperlink{class_q_c_p_item_line_ae184140b61b2ef5b8edde76304447200}{QCPItem\+Line\+::draw}} function doesn\textquotesingle{}t need to care about setting any antialiasing states, because the default antialiasing hint is already set on the painter when the \mbox{\hyperlink{class_q_c_p_grid_aad9a312d998e2d170956334d4cb80be2}{draw}} function is called, and that\textquotesingle{}s the state it wants to draw the line with.
